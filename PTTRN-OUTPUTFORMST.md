# Analyze Output Format
- If you see this output, then:

**List of Lists:** 
- Backtracking is almost always the answer
- Generate all possibilities
- Use recursion with choice/no-choice pattern

**Single Number (max/min profit, cost, ways, jumps):**
- dynamic programming for optimization
- greedy for local optimal choices
- mathematical approach for counting

**Modified Array/String (in-place operations):**
- Two pointers for in-place modifications

**Ordered List (sorted sequence, valid task order):**
- Sorting with custom comparators
- Topological Sort Of Dependencies
- Heap for maintaining order

# Keyword Pattern Recognition

**Dynamic Programming Keyword:**
- "Number Of Ways"
- "Maximum/Minimum" + "sum/profit/cost"
- "Can you reach"
- "longest/shortest subsequence"
- "Optimal" or "best"

**Two Pointers Keywrods:**
- "Palindrome"
- "Sorted Array"
- "Target Sum"
- "Remove Duplicates"

**Heap Keywords:**
- "K largest" or "K smallest"
- "Top K elements"
- "Median"
- "Priority"

**Stack Keywords:**
- "Parentheses" or "Brackets"
- "Valid expression"
- "Nested Structure"
- "Undo Operations"

**Monotonic Stack Keywords
- "Next greater element"
- "Next smaller element"

**HashMap Keywords**
- "Count Frequency"
- "Find Duplicates"
- "Anagram"

**Trie Keywords**
- "Word Search"
- "Word prefixes"

**Greedy Keywords:**
- "Minimum Operations"

**Union Find Keywords**
- "Connected Components"
- "Number Of Groups"

**Binary Search Keywords:**
- "Kth element"
- "Search in sorted"
- "Minimize maximum"
- "First/last occurence"

**Sliding Window:**
- "Substring with conditions
- "SubArray" with fixed/variable size
- "Maximum/minimum window"
- "Contains all"
